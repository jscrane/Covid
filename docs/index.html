Covid-19 in Ireland

Introduction

We perform simple mathematical modelling from government-published data, of recorded cases of Covid-19 in the Republic of Ireland.

For a refresher on exponential growth and epidemics, please see the 3Blue1Brown video.

The Data

Data is published daily at an Irish Government website and manually entered into a data-file. We use only the daily increases in cases and deaths.

d <- read.csv(file="cases.csv", header=TRUE, sep=",")
d[is.na(d)] <- 0
d$date <- as.Date(d$date)
d$day <- as.integer(d$date - d$date[1])
d$total_cases <- cumsum(d$cases)
d$total_deaths <- cumsum(d$deaths)
ld <- tail(d, 1)
paste(ld$date, "total:", ld$total_cases, "cases,", ld$total_deaths, "deaths")
## [1] "2020-03-22 total: 907 cases, 4 deaths"

Exponential Phase

From the plot below, it is clear that as of 19 March, we are currently in the exponential-growth phase. (An r-squared value of 0.97 indicates a very good fit.)

f <- log10(total_cases) ~ date
plot(f, data=d, ylab="log(cases)", main="Confirmed cases")
m <- lm(f, data=d)
abline(m)
r2 <- summary(m)$r.squared
dbl <- log10(2)/coef(m)[2]
wx <- 2**(7 / dbl)
legend("bottomright", bty="n", legend=c(paste("r^2:", format(r2, digits=2)), paste("dbl:", format(dbl, digits=2), "days"), paste("wx:", format(wx, digits=2))))

The parameter \(dbl\) in the plot is the doubling time in days, computed from the best-fit slope (cf. the rule of 70). The parameter \(wx\) is the weekly growth factor.

We can predict the number of cases at the end of the month and see it is in line with estimates from the government.

pd <- as.Date("2020-03-30")
p <- 10**(predict(m, newdata=data.frame(date = pd)))
paste(pd, "predict", format(p, digits=2), "cases")
## [1] "2020-03-30 predict 14153 cases"

Notes

Becoming a statistic on this graph requires time and effort:

  • call a GP
  • convince them of symptoms
  • be tested by appointment, either at home, or at a test centre
  • wait for result

It is not clear what the average start-to-finish time is. However, the PCR test can take up to 2 days; and on 22 March, the HSE said that the average waiting time to get a test was 4-5 days (and there was a backlog of 40000).

Statistics on testing are only published weekly. However, we can get a rough estimate by combining the latest figure with the latest number of cases:

t <- read.csv(file="testing.csv", header=TRUE, sep=",")
t$date <- as.Date(t$date)
m <- merge(d, t, by="date")
lm <- tail(m, 1)
lt <- ld$total_cases * lm$tests / lm$total_cases
paste(ld$date, "positive rate:", format(lm$total_cases / lm$tests, digits=2), "tests:", format(lt, digits=2), "(estimated)")
## [1] "2020-03-22 positive rate: 0.044 tests: 20613 (estimated)"

(This assumes that the rate of positives remains constant over the interval since the test figures were published.)

From all this, it is possible that the testing system has been saturated and this is limiting the number of cases reported per day around this time. (This can be seen in a flattening of the graph above.)

If 4% of the backlog tested positive, the number of confirmed cases on the 22 March would have been 2500. Furthermore, the cases actually confirmed on a given day could be up to a week old.

Logistic Phase

In the real world, exponentials don’t continue indefinitely; after some time, they flatten into a logistic (sigmoid) curve.

d$lag.cases = c(NA,head(d$cases, -1))
d$growth = d$cases / d$lag.cases
plot(growth ~ date, data=d)

The exponential phase ends when the growth rate slows to 1. (However this is not currently the case: the data is too noisy.)
